Design notes — remove from production

Typography & spacing

  • font-family on the table is --ff-mono at 12px — exactly matching .surface-table. The chips drop to 10px so 2 chips fit in the same vertical rhythm as a single mono number.
  • Row padding is 8px vertical (vs. 5px on Surface) — Sessions rows carry more chips, so the extra breathing room prevents chip stacks from feeling cramped. The cost cell uses inner 4px 8px so the magnitude bar has its own micro-padding.
  • letter-spacing: -0.01em on the cost value is the same trick used by KPI .v — tightens the dollar number so $5.8210 reads as one glyph cluster.

Why this column layout

  • Token columns documented via help-pop, not super-header. Earlier draft used a visible Tokens column-group label in peri above the 4 numeric columns. Per review, replaced with a single ? help-pop on the Input header documenting all 4 token types (input / output / cache read / cache create) and their relative pricing. The popup carries far more useful information than the super-label could (cache R is ~1/10 input price, cache C is 1.25× input, etc.) and matches the existing .help-host/.help-btn/.help-pop grammar already used elsewhere on the dashboard. Cache R/Cache C are still dimmed (--ink-faint) at the column level so they read as supporting evidence rather than headline metrics.
  • Service tier also gets a help-pop. priority / standard / batch are not self-explanatory — a user seeing all 3 chips on the same row needs to know that priority means a paid latency-guarantee tier (Anthropic-side billing concept, mixable per-message via /model switch / reconnection), not a Claude Code preference.
  • Session ID column dropped per maintainer feedback. The 8-char shortened ID was rarely actionable in the row-level ledger view: project + 開始時刻 already uniquely identifies a session at a glance, and freed width goes to the Project column where long names actually need it. Drill-down to per-message timeline (where the full session_id would be shown in a header) is a future feature.
  • Cost gets the headline treatment. Inline magnitude bar with --cost-pct custom-property scaling (set per row at render time as max(row_cost) / row_cost). The whale row visually anchors the table at 100%; everything else is a fraction. Whale + cheap rows get tinted values (peach / ink-soft) so cost-class is recognizable in <200ms.
  • Models / Tier are chip-stacks, not strings. Color-coding is intentional and reuses the dashboard's accent palette so cost-class is doubly encoded: opus reads as coral (premium) without needing to know the price table, sonnet reads as mint (workhorse), haiku as peach (cheap-and-fast). Tier reuses peri (priority) / ink-soft (standard) / peach (batch) — peri was chosen for priority instead of mint to avoid color-clash with the active-row mint background tint.
  • 進行中 pill reuses conn-pulse (already in 10_components.css) so live sessions inherit the same heartbeat motif as the connection badge. Active rows also get a subtle mint-tinted background (rgba(111,227,200,0.04)) so they pop without needing to be visually heavy.

What was sacrificed

  • Per-message granularity. The chip opus (8) shows count of llm_response events but not their individual cost contribution. Click-through to a per-session detail panel is the natural next step (out of scope for this mock).
  • Sortability indicator. No header sort glyphs in this draft — the spec says "sorted by start time descending" only. If the maintainer wants ad-hoc sort by cost / duration / token-count, add aria-sort + a small chevron in column headers. Cheap to add later.
  • Filter UI. No project-filter dropdown / model-filter chips above the table. The Period toggle pattern from Overview/Patterns could be reused for "今日 / 7d / 30d" but that pushes scope past Issue #103. For 20 rows, scrolling is faster than filtering anyway.
  • モデル使用比率のドーナツ/円グラフは、本ページではなく Overview ページの「モデル分布」パネル (既存「プロジェクト分布」横の stacked bar grammar を流用) として別 Issue で追加予定。Sessions ページは「1 行 = 1 session の台帳」が役割なので、集約系のチャートは持たない。

What changes at narrow viewport

  • At <1280px, the two cache-token columns (Cache R / Cache C) hide via display: none on col and th/td. The full numbers stay in the API response, just not in the visual table — they're recoverable via row hover tooltip (proposal: data-tip pattern from 20_help_tooltip.css, not implemented in this mock).
  • At <1080px, recommend collapsing chip stacks to "most-frequent label only" (e.g. opus 8 instead of opus 8 / sonnet 2) and showing the secondary chip in a hover tooltip. Not implemented — would need JS.
  • At <720px, the table becomes scrollable horizontally with a sticky Session + Project column-pair. CSS-only via position: sticky; left: 0 on the first 2 cells. Also not implemented in this mock.

Class additions vs reuse summary

  • REUSED .panel, .panel-head, .help-host/.help-btn/.help-pop, .kpi/.kpi-row, .page-nav, conn-pulse keyframes
  • EXTENDED .surface-table grammar (mono / 12px / num-right / hover-row) — namespaced under .sessions-table because of the colgroup + group-header row
  • NEW .model-chip.m-{opus,sonnet,haiku} — mirrors .mode-chip family
  • NEW .tier-chip.t-{priority,standard,batch} — same pattern
  • NEW .cost-cell + .cost-bar — inline magnitude visualisation, scales via --cost-pct CSS var set per-row by aggregator
  • NEW .live-pill — wraps conn-pulse in a pill shape for the 進行中 marker

Open questions for the maintainer

  • Whale-row peach left-edge (3px inset box-shadow) — too loud? Could be dialled down to a 1px border instead, or removed entirely if the cost-bar magnitude is sufficient.
  • Cost bar gradient goes mint-soft → peach → coral. An alternative is a single solid peri bar (matches the help-pop trigger color). The gradient encodes "cheap → expensive" as a bonus; flat peri reads as "neutral magnitude bar". Either works; kept the gradient for now because the headline-cost framing in the spec asks for visual prominence.
  • Row tinting for active sessions — currently rgba(111,227,200, 0.04) (very subtle). At very narrow viewports / on glossy displays this can drop below perceptible threshold. Consider bumping to 0.06 or pairing with a left-edge mint stripe to mirror the whale treatment.
